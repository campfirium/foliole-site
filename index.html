Design principles

Open code, open data, local-first ownership.

Open source

The code is open source, so anyone can review, build, and improve the tool.

Open data

Foliole uses SQLite and provides a Markdown mirror for reading, migration, and reuse.

Local first

No account system. No central cloud sync. Your devices sync over the local network.

Core features

Built for the whole reading loop.

Native incremental reading

Extract passages, create cloze deletions, and refine material without leaving the reading flow.

Integrated FSRS scheduling

Review items are scheduled with FSRS, an open and efficient spaced repetition scheduler.

Bring sources together

Work with local files, web documents, Obsidian notes, and Readwise Reader exports.

Index external documents

Search and view local folders without copying or moving the original files.

Complex content support

Read Markdown, PDF, EPUB, LaTeX math, code blocks, and other structured content.
